# File Type Rules

## 总体目标

- 扫描目标文件中的中文
- 生成对应的 i18n key
- 替换源码中的中文
- 写入语言包

## JS / TS

- 使用项目约定的 `t(...)`、`$t(...)` 或 `useI18n().t(...)`
- 替换源码中的中文字符串

## Template

- 使用模板语法的国际化调用
- 处理文本节点、属性、绑定表达式
- 替换源码中的中文文本和属性值

## 静态文件

- 按项目约定处理
- 不要假设所有静态文件都能直接做 i18n 替换

## 语言包

- 更新项目原有语言包目录
- 不要强制固定目录名
- 将多语言文案写入语言包
